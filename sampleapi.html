Sample app using the Social Stream HTTP/WSS remote control API

Ensure that you have matching session values, both with this page if testing (for example, dock.html?session=xxxx&server), or used when calling the API itself.

See this link for details on how to add these commands to your stream deck or application.


As a basic example though, a call that uses a basic GET request would be structured like:

https://io.socialstream.ninja/{sessionID}/{action}/{target}/{value}.
